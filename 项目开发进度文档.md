# AlphaSignal 项目开发进度文档

## 📅 2026-02-14 开发进展记录

### 1. ALPHA 基金模块 (iOS 原生端) - **核心功能闭环**
*   **实时估值引擎**：实现了基于 `ValuationActor` (Swift 6) 的高性能推算逻辑，支持 REST/SSE 双通道同步。
*   **精算指标体系**：同步了 Web 端的夏普比率 (Sharpe)、最大回撤 (MaxDD) 及其 S/A/B/C 评级系统。
*   **情报关联引擎**：开发了 `IntelligenceLinkageEngine`，实现了基于持仓关键词在本地 `SwiftData` 中的模糊检索与解释闭环。
*   **智能预警**：集成了 2σ 统计学异常波动报警及 iOS **Critical Alerts (紧急警报)**。
*   **UI/UX 增强**：实现了 `LiquidTicker` (数字跳动动画) 和 `FundSparkline` (微型走势图)。

### 2. Backend V1 生产级架构重构
*   **基础设施**：
    *   引入了 **SQLModel (SQLAlchemy 2.0)** 作为核心 ORM，建立强类型数据契约。
    *   部署了 **Redis Pub/Sub** 作为实时广播中枢，彻底解耦计算与推送。
*   **BFF (Backend For Frontend) 模式**：
    *   **iOS BFF (`/api/v1/mobile`)**：针对移动端进行字段裁剪和聚合优化。
    *   **Web BFF (`/api/v1/web`)**：保持高密度数据输出，兼容 Plotly 深度分析。
*   **行情逻辑下沉**：将 **Gold Spread (内外盘价差)** 的复杂精算逻辑从 Node.js 层下沉至 Python `MarketService`。

### 3. Web 端 V1 适配与 Bug 修复
*   **全量接口迁移**：将 Web 端的数据获取、历史对账、情报检索及实时流全部切换至 V1 BFF。
*   **Bug 修复**：
    *   修复了“板块归因”Treemap 权重显示为 `-` 的数据绑定错误。
    *   修复了“对账监控”页面因 `Decimal` 序列化导致的 `toFixed` 崩溃问题。
    *   统一了全端日期格式为标准 ISO8601 (带毫秒及 Z 后缀)。

### 4. 当前系统状态
*   **后端**：V1 架构已上线，兼容 Legacy 接口。
*   **iOS**：V1 路由已全量接入，进入精算级调试阶段。
*   **Web**：完成 V1 迁移，性能与稳定性大幅提升。
